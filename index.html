Researching 100 Apps as Agent-Callable Tools

An automated intelligence pipeline executing web searches and LLM extraction—not a manual spreadsheet.

Key Patterns & Findings

Authentication Dominance

87/100 apps support API key auth, and 82/100 support OAuth2. Most self-serve platforms provide both, making agent tooling highly standardized.

Self-Serve is the Norm

Over 95% of apps are completely self-serve. Genuinely gated apps are the exception (e.g., PitchBook is gated-paid, WhatsApp Business and Consensus are gated-approval).

The Discoverability Hurdle

The "AI, Research and Media-native" category initially looked blocking (half appeared not-buildable). Verification proved the real issue was docs discoverability for newer tools, not API gating.

Mainstream Wins

CRM, Communications, Dev/Infra, and Productivity categories were 10/10 buildable-now. These established markets have mature, self-serve, and highly discoverable APIs.

The Agent Architecture

This dataset was generated using a custom automated pipeline that searches for documentation, fetches the HTML, and uses an LLM (Groq / Llama 3.3 70B) to extract structured JSON against a strict Pydantic schema.

GitHub Repository: https://github.com/Yashpreetg24/Composio-Assignment

A note on process honesty: The initial run failed on 100% of rows due to an empty Anthropic API credit balance. The pipeline caught the error gracefully. On the second run (switched to Groq), it succeeded on 95/100. The remaining 5 rows failed strict Pydantic enum validation ("unknown" instead of allowed values), which prompted the addition of a resilient retry loop.

The Verification Loop

To ensure accuracy without hallucination, the pipeline utilized a 3-layer verification process.

Layer 1: First Pass

The agent confidently classified 83/100 apps. 17 apps were flagged with low confidence or lacked concrete evidence URLs.

Layer 2: Automated Advanced Recheck

An automated script re-ran the 17 flagged apps with advanced URL guessing (e.g., trying /docs, /api) and secondary specific search queries. 6/17 verdicts were corrected, 2/17 confidence levels improved, leaving 9 unresolved.

Layer 3: Human Spot-Check

Sampled flagged apps (Attio, Plain, Devin, Fathom). The automated corrections for Attio and Plain were confirmed accurate. Devin and Fathom were STILL WRONG even after two automated passes—both actually have real, documented APIs that the agent failed to discover. These two were corrected manually.

Final Unresolved

7 apps remain honestly flagged for human review (Pylon, systeme.io, fanbasis, MrScraper, iPayX, Consensus, higgsfield). No answers were invented; insufficient evidence was found even after extensive passes.
